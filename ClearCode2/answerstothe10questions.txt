1. Проблемы с именами

task → Task
d → description
p → priority
f → sortTasksByPriority
cnt → taskCount
list → tasks
2. SRP нарушен
В main всё сразу: ввод, сортировка, вывод, работа с памятью. Нужно разбить на классы и функции.

3. Дублирования кода
Ввод/вывод задачи повторяются. Сортировка написана вручную вместо std::sort.

4. Улучшение структуры
Выделить класс Task, класс TaskManager и отдельные функции для ввода.

5. При некорректном вводе

Приоритет вне 1–5: программа примет любое число, сортировка сломается
Описание длиннее 199 символов: обрежется без предупреждения
6. Опасности using namespace std и char d[200]

using namespace std — конфликты имён
char d[200] — переполнение буфера при вводе >199 символов
7. Какой алгоритм из STL лучше
std::sort с лямбдой для сравнения по приоритету.

8. Утечки памяти
В исходном коде нет (есть delete[]), но риск есть. Решение: std::vector.

9. Поведение при cnt == 0 или cnt == 1

cnt == 0: выделяет нулевой массив (странно, но работает)
cnt == 1: сортировка не нужна
10. Как проверить сортировку
Написать юнит-тест или вручную ввести задачи с приоритетами 2,5,1 и проверить вывод 5,2,1.
